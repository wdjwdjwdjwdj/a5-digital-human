📋 列表页原型

景点列表页 - 组件状态与交互流程

原型说明

  • 展示列表页的6种核心状态：默认、悬停、点击、加载中、空状态、错误状态
  • 包含所有交互组件的视觉呈现和状态变化
  • 标注了详细的交互说明和用户体验注意事项
  • 采用 ardot 设计风格，简洁现代，注重可用性

📝 交互说明注释

1. 列表项交互流程：

• 默认状态 → 鼠标悬停 → 视觉反馈（边框、阴影、背景）

• 悬停状态 → 鼠标点击 → 缩放动画 → 跳转到详情页

• 整个交互过程应有 200ms 的过渡动画，保证流畅性

2. 加载状态处理：

• 首次加载显示骨架屏或加载动画

• 下拉刷新时显示顶部加载指示器

• 上拉加载更多时显示底部加载提示

3. 空状态设计原则：

• 使用友好、易懂的图标和文字

• 提供明确的下一步操作指引

• 避免使用技术性术语或错误代码

4. 错误状态恢复机制：

• 明确告知错误原因（网络、权限、数据不存在等）

• 提供可操作的恢复按钮（重试、返回、联系客服等）

• 记录错误日志，便于后续排查问题
