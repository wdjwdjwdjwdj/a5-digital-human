# A5-景区导览服务AI数字人 — 双端UI设计规范

## 一、项目概述

**项目名称**：2026中国软件杯赛题-A5-景区导览服务AI数字人
**目标平台**：移动端（小程序/APP）+ Web端（管理后台/大屏展示）
**核心功能**：AI数字人导览、景区地图导航、语音讲解、路线规划、景点推荐

---

## 二、设计方向与风格定位

### 2.1 整体风格

**"科技国风"** — 将中国传统美学与现代科技融合

- **视觉基调**：以中国传统色彩为灵感（黛青、朱砂、牙白、墨黑），融入现代玻璃拟态（Glassmorphism）效果
- **情感传达**：文化底蕴 + 科技未来感 + 亲切友好
- **设计参考**：故宫博物院小程序的国风美学 + 文心一言的简洁科技感

### 2.2 色彩系统

```css
:root {
  /* 主色调 - 黛青 */
  --primary: #2B5F75;
  --primary-light: #4A8FA8;

  /* 辅助色 - 朱砂 */
  --accent: #C73E3A;

  /* 背景色 */
  --bg-primary: #F5F0E8;    /* 牙白 */
  --bg-secondary: #FFFFFF;

  --text-primary: #1A1A1A;
  --text-secondary: #666666;

  /* 功能色 */
  --success: #4CAF50;
```

### 2.3 字体规范

- **中文标题**：思源宋体 / 方正清刻本悦宋（有文化感）
- **中文正文**：思源黑体 / 苹方
- **英文/数字**：Playfair Display（标题）+ Source Sans Pro（正文）
- **字号规范**：
  - 大标题：28-32px
  - 小标题：20-24px
  - 正文：16px
  - 辅助文字：12-14px

## 五、核心组件设计

### 5.1 AI数字人组件

**展示形式**：
- **2D版本**：Live2D风格，轻量化，适合移动端
- **3D版本**：WebGL渲染，支持360°旋转，适合Web端大屏

**交互状态**：
- 待机：呼吸动画，偶尔眨眼
- 聆听：头部微倾，出现声波可视化
- 思考：出现思考气泡/加载动画
- 说话：嘴部同步，手势配合
- 情绪变化：开心、疑惑、专注等表情

### 5.2 地图组件

**功能**：
- 自定义地图样式（国风配色）
- 景点标记（支持分类图标）
- 路线规划（步行/观光车）
- 实时定位
- AR实景导航入口

### 5.3 语音交互组件

**界面元素**：
- 麦克风按钮（按住说话）
- 波形可视化（录音中）
- 语音转文字实时显示
- 语音播放进度条

## 六、动画与动效

### 6.1 页面过渡

- 页面切换：滑动过渡（移动端）/ 淡入淡出（Web端）
- 元素加载：骨架屏 → 内容淡入
- 下拉刷新：国风加载动画（如：旋转的祥云）

### 6.2 微交互

- 按钮点击：缩放0.95 + 涟漪效果
- 卡片悬停：上浮 + 阴影加深
- 数字变化：数字滚动动画
- 语音输入：波形跳动动画

### 6.3 数字人动画

- 入场：从底部滑入 + 淡入
- 说话：嘴部Blend Shape动画
- 手势：配合语言内容的预设手势
- 情绪：面部微表情变化

## 八、技术实现建议

### 8.1 前端技术栈

- **移动端**：微信小程序 / React Native / Flutter
- **Web端**：React / Vue + TypeScript
- **UI组件库**：TDesign / Ant Design Mobile
- **地图**：高德地图SDK / 腾讯地图SDK

- **大模型**：文心一言 / 通义千问 / GPT-4
- **数字人驱动**：Live2D / VRM / 自研

## 十、总结

本设计规范以"科技国风"为核心，将中国传统文化美学与现代AI技术完美融合。通过统一的色彩系统、组件规范和交互模式，确保移动端与Web端的一致性和高品质用户体验。AI数字人作为核心亮点，需要在视觉表现和交互体验上做到行业领先，为游客提供沉浸式的智能导览服务。
